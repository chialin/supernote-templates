# 產品需求文件：基礎橫線筆記本樣板

## 簡介/概述

建立一個簡潔、實用的橫線筆記本樣板，適用於 Supernote 電子筆記裝置（Nomad 和 Manta）。此樣板提供類似傳統紙本筆記本的橫線格式，讓使用者可以在上面自由書寫各種類型的筆記內容。

本樣板的設計理念是「極簡主義」——只提供基本的橫線，不包含任何額外的欄位或裝飾元素，讓使用者擁有最大的書寫自由度。樣板將利用專案現有的多裝置支援系統，自動適配不同尺寸的 Supernote 裝置。

## 目標

1. **提供通用筆記解決方案**：建立一個適用於各種筆記場景的基礎樣板（會議記錄、日記、課堂筆記等）
2. **確保多裝置一致性**：在 Nomad 和 Manta 裝置上都能正確顯示，橫線密度和視覺效果保持一致
3. **最大化書寫空間**：充分利用裝置的安全區域，提供盡可能多的書寫空間
4. **建立簡單範例**：為未來的樣板開發者提供一個最簡化的參考實作

## 功能需求

### FR-1：橫線顯示
系統必須在頁面上顯示均勻間距的黑色實線橫線。

### FR-2：行數標準
在 Nomad 裝置上，樣板必須顯示 18 條橫線，每條線高度為 94px。

### FR-3：多裝置支援
樣板必須在 Nomad (1404×1872px) 和 Manta (1920×2560px) 兩種裝置上正確顯示。

### FR-4：安全區域遵守
橫線必須在裝置的安全區域內繪製，避免被 Supernote 系統工具列遮擋。

### FR-5：全頁填充
橫線必須填滿整個安全區域的可用高度，充分利用書寫空間。

### FR-6：視覺一致性
橫線的粗細、顏色和間距在兩種裝置上必須保持視覺一致性（考量到相同的 300 DPI）。

## 非目標（超出範圍）

1. **不包含頁首元素**：不提供日期欄位、標題欄位或任何頁首區域
2. **不包含裝飾元素**：不添加頁碼、圖示或其他視覺裝飾
3. **不提供多種樣式**：不提供不同行距或橫線粗細的變化版本
4. **不包含左側邊界**：不預留傳統筆記本的裝訂邊或左側留白
5. **不支援其他裝置**：目前僅支援 Nomad 和 Manta，不考慮其他尺寸的裝置

## 設計考量

### 視覺設計
- **橫線樣式**：黑色實線
- **橫線粗細**：使用標準 1px 或 2px（以確保在 e-ink 螢幕上清晰可見）
- **行高**：Nomad 為 94px（18 行），Manta 將依比例自動計算
- **背景**：純白色背景

### 使用者體驗
- 橫線間距應該適合一般手寫字體大小
- 橫線應該明顯但不過於突兀，不干擾書寫體驗
- 整體視覺應該乾淨、簡潔

## 技術考量

### 整合現有系統
1. **使用 CSS 變數系統**：利用 `styles/devices.css` 中定義的裝置變數和安全區域設定
2. **響應式設計**：使用媒體查詢自動適配不同裝置尺寸
3. **自動填充機制**：參考 `auto-fill-rows.js` 的實作方式，使用動態生成方式填充橫線至可用空間

### 檔案結構
- 樣板檔案位置：`templates/[template-name].html`
- 輸出位置：透過 `npm run generate` 自動產生至 `dist/nomad/` 和 `dist/manta/`

### 技術限制
- 必須遵守 Supernote 的安全區域設定（上 30px、右 30px、下 150px、左 150px）
- 輸出格式必須為 PNG
- 解析度必須精確符合裝置像素尺寸

## 驗收標準

### AC-1：視覺正確性
- [ ] 在 Nomad 裝置上顯示 18 條清晰的黑色橫線
- [ ] 在 Manta 裝置上顯示相同視覺密度的橫線
- [ ] 橫線間距均勻一致

### AC-2：空間利用
- [ ] 橫線填滿整個安全區域的可用高度
- [ ] 第一條線不會太靠近上邊緣
- [ ] 最後一條線不會被工具列遮擋

### AC-3：多裝置一致性
- [ ] 在兩種裝置上都能正確生成 PNG 檔案
- [ ] 兩種裝置上的視覺效果保持一致（考量 DPI）

### AC-4：可用性
- [ ] 使用者可以在橫線上清晰書寫
- [ ] 橫線不會過於搶眼，影響書寫內容的可讀性

### AC-5：技術實作
- [ ] 使用 `npm run generate` 可以成功產生兩種裝置的 PNG 檔案
- [ ] 檔案大小合理，不會過大
- [ ] 符合專案的程式碼規範和架構模式

## 開放問題

1. **橫線粗細最佳值**：需要在實際裝置上測試，確認 1px 或 2px 哪一個在 e-ink 螢幕上效果更好
2. **邊距微調**：首次生成後需要確認橫線是否太靠近邊緣，可能需要微調安全區域設定
3. **Manta 行數**：需要確認 Manta 裝置上自動計算的行數是否符合預期（預計約 24-25 行）
